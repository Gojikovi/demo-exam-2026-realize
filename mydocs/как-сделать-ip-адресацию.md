### Условие:

На всех устройствах необходимо сконфигурировать IPv4:

- IP-адрес должен быть из приватного диапазона, в случае, если сеть локальная, согласно RFC1918

- Локальная сеть в сторону HQ-SRV(VLAN 100) должна вмещать не более 32 адресов

- Локальная сеть в сторону HQ-CLI(VLAN 200) должна вмещать не менее 16 адресов

- Локальная сеть для управления(VLAN 999) должна вмещать не более 8 адресов

- Локальная сеть в сторону BR-SRV должна вмещать не более 16 адресов

- Настройте интерфейс, в сторону HQ-RTR, интерфейс подключен к сети 172.16.1.0/28

- Настройте интерфейс, в сторону BR-RTR, интерфейс подключен к сети 172.16.2.0/28

### ISP

Адреса сетей нам уже дают. Поэтому просто изменяем последний октет в диапазоне от 1 до 14.

ISP -> HQ-RTR: 172.16.1.1/28

ISP -> BR-RTR: 172.16.2.1/28

### HQ-RTR

В сторону ISP используем адрес сети 172.16.1.0/28

HQ-RTR -> ISP: 172.16.1.2/28

Дальше рассчитываем для VLAN. Воспользуемся сайтом [IPmeter](https://ipmeter.ru/)

> [!NOTE]
> Если в условий написано "не менее N адресов", действуем следующим образом. Ниже показано и расписано

Будем использовать привычные частные ip-адреса из диапазона 192.168.0.0 — 192.168.255.255

В поле "Исходная сеть" впишем например 192.168.1.0 (третий октет - любой удобный для вас). Маску оставим /24. "Размеры подсетей" - 30, 16 и 6 исходя из условия (для VLAN 100 и 999 вычитаем широковещательный адрес и адрес сети). 

Для VLAN 200 мы не вычитаем широковещательный адрес и адрес сети при расчёте через ipmeter. Тем самым, мы обеспечваем минимальное количество адресов, которое должно быть в подсети

В размер подсетей мы задаём в ipmeter количество ip-адресов на хосты, а в условии имеется ввиду общее число ip-адресов (хосты + широковещательный + ip-адрес сети)

### BR-RTR

В поле "Исходная сеть" впишем например 192.168.2.0 (третий октет - любой удобный для вас). Маску оставим /24. "Размеры подсетей" - 14 исходя из условия (вычитаем широковещательный адрес и адрес сети)

Нажимаем "Вычислить". Получили табличку для BR-сети (BR-SRV). Можно посчитать вручную самим.

### HQ-SRV

Изменяем последний октет из расчитанной сети VLAN 100

Например: 192.168.1.2/27

Шлюзом по умолчанию будет IP-адрес на интерфейсе HQ-RTR в сторону HQ-SRV VLAN 100 : 192.168.1.1

### HQ-CLI

Изменяем последний октет из расчитанной сети VLAN 200

Например: 192.168.1.34/27 (учитываем что IP-адрес будет получен по DHCP из диапазона 192.168.1.34 - 192.168.1.62)

Шлюз по умолчанию придёт по DHCP (ip-адрес роутера в сторону HQ-CLI VLAN 200): 192.168.1.33

### BR-SRV

Изменяем последний октет из рассчитанной BR-сети

Например: 192.168.2.2/28

Шлюз по умолчанию (ip-адрес роутера HQ-RTR): 192.168.2.1 

IP-адресация готова. Останется занести данные в таблицу отчёта.
